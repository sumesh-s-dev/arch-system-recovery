# arch-system-recovery v1.1.1

## Highlights

- Added explicit `--boot` support for systems with a separate `/boot` partition.
- Strengthened root, `/boot`, and EFI auto-detection using mounted `fstab` ownership checks.
- Updated diagnose and health-check flows to mount separate `/boot` layouts read-only when needed.
- Routed TUI operations through the canonical CLI flow so the same safety checks apply everywhere.
- Added verified self-update bundles and `make dist` release packaging.

## Safety and Reliability Fixes

- Fixed release bundle naming so published assets match the updater contract.
- Auto mode now fails fast instead of prompting when bootloader detection is unknown.
- Fixed the `wpa_supplicant` fallback temp-file path and tightened temp-file permissions.
- Snapshot rollback now rejects unsafe snapshot names before touching the filesystem.

- `bash tests/run_tests.sh`
