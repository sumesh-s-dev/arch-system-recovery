# arch-system-recovery v1.1.2

## Highlights

- Moved runtime logs to a private per-session directory under `/tmp/arch-recovery-session.*`.
- Added signed release manifest verification for self-update using `ssh-keygen`.
- Hardened the release pipeline to verify signed tags and signed manifests before publishing.
- Made release bundles reproducible so signed manifests verify cleanly in CI and on GitHub Releases.
- Added a privileged loopback integration suite covering ext4, BTRFS, LUKS, LVM, and boot repair paths.

## Security and Reliability Fixes

- Removed predictable installer temp files in favor of `mktemp`.
- Stopped passing Wi-Fi passwords on the command line for `nmcli` and `iwctl`.
- Kept checksum verification, then layered signed-manifest verification on top for authenticated releases.

## Verification

- `make check`
- `bash tests/run_tests.sh`
- `make dist`
- `make release-manifest`
